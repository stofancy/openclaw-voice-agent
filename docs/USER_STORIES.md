## 1. 用户故事 (User Stories)

### 1.1 Epic: 连接管理

#### US-01: 打开网页自动连接

```
作为用户，
当我打开语音助手网页时，
我希望系统自动建立 WebRTC 连接，
以便能够进行语音对话。

验收标准:
- [ ] 网页加载完成后 2 秒内自动尝试连接
- [ ] 连接成功时显示"已连接"状态
- [ ] 连接失败时显示错误信息，3 秒后自动重试
- [ ] 重试失败显示"连接失败，请点击重试"按钮
```

#### US-02: 手动重连

```
作为用户，
当连接断开时，
我希望点击按钮能手动重连，
以便快速恢复对话。

验收标准:
- [ ] 点击"重连"按钮后开始连接
- [ ] 重连期间显示"正在连接..."状态
- [ ] 重连成功显示"已连接"
- [ ] 重连失败显示错误信息
```

---

### 1.2 Epic: 语音对话

#### US-03: 开始录音

```
作为用户，
当我点击"开始对话"按钮时，
我希望系统开始采集我的麦克风音频，
以便开始与 AI 对话。

验收标准:
- [ ] 首次点击时请求麦克风权限
- [ ] 权限拒绝时显示"需要麦克风权限才能使用语音助手"
- [ ] 权限同意后开始采集音频
- [ ] 状态显示"正在录音..."
```

#### US-04: 语音活动检测

```
作为用户，
当我停止说话时，
希望系统能自动检测到并结束录音，
以便不需要手动停止。

验收标准:
- [ ] 检测到静音后 1 秒自动停止录音
- [ ] 停止录音后自动发送音频到服务器
- [ ] 静音时间 < 1 秒时继续录音
- [ ] 手动点击"停止"按钮立即停止录音
```

#### US-05: 接收 AI 回复

```
作为用户，
当 AI Agent 回复时，
我希望听到 AI 的语音回复，
以便完成对话交互。

验收标准:
- [ ] 收到 AI 回复后自动开始播放语音
- [ ] 播放期间显示"正在播放..."
- [ ] 播放完成后状态恢复"等待中"
- [ ] 播放失败显示错误信息
```

#### US-06: 多轮对话

```
作为用户，
当 AI 说完后，
我希望继续说话进行下一轮对话，
以便进行连续的交流。

验收标准:
- [ ] AI 播放完成后自动准备下一轮录音
- [ ] 可以连续进行多轮对话
- [ ] 每轮对话独立处理
- [ ] 不会因为上一轮错误影响下一轮
```

### 1.3 Epic: 状态显示

#### US-07: 显示连接状态

```
作为用户，
我希望看到当前的连接状态，
以便我知道是否可以说话。

验收标准:
- [ ] 初始显示"等待连接"
- [ ] 连接中显示"正在连接..."
- [ ] 已连接显示"已连接"
- [ ] 断开显示"连接断开"
```

#### US-08: 显示对话状态

```
作为用户，
我希望看到当前的对话状态，
以便我知道是否可以说话。

验收标准:
- [ ] 空闲显示"等待中"
- [ ] 录音显示"正在录音..."
- [ ] 等待回复显示"正在思考..."
- [ ] 播放显示"正在播放..."
```

#### US-09: 显示音量

```
作为用户，
我希望看到当前的音量大小，
以便我知道系统是否正在采集我的声音。

验收标准:
- [ ] 录音时显示音量波形/条
- [ ] 静音时音量条消失或归零
- [ ] 音量条延迟 < 100ms
```

---

### 1.4 Epic: 错误处理

#### US-10: 麦克风权限错误

```
作为用户，
当麦克风权限被拒绝时，
我希望看到友好的错误提示，
以便我知道如何解决。

验收标准:
- [ ] 显示"需要麦克风权限"提示
- [ ] 提供"打开设置"按钮
- [ ] 提供"重试"按钮
```

#### US-11: 网络断开

```
作为用户，
当网络断开时，
我希望看到明确的提示，
以便我知道对话已中断。

验收标准:
- [ ] 断开时显示"网络断开"
- [ ] 自动尝试重连（最多 3 次）
- [ ] 重连失败显示"连接失败，请检查网络"
- [ ] 提供"重试"按钮
```

#### US-12: AI 服务错误

```
作为用户，
当 AI 服务出错时，
我希望看到错误提示并可以重试。

验收标准:
- [ ] 显示"AI 服务暂时不可用"
- [ ] 提供"重试"按钮
- [ ] 错误不影响已录音的音频
```
