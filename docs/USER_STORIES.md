## 1. 用户故事 (User Stories)

### 1.1 核心功能

| # | 用户故事 | 优先级 |
|---|---------|--------|
| US-01 | 作为用户，我希望打开网页就能通过麦克风和 AI Agent 对话 | P0 |
| US-02 | 作为用户，我希望说话时 AI 能实时识别我的语音并回复 | P0 |
| US-03 | 作为用户，我希望 AI 的回复能直接转换成语音播放给我听 | P0 |
| US-04 | 作为用户，我希望能看到当前是"正在录音"还是"正在说话"的状态 | P1 |
| US-05 | 作为用户，我希望网络断开后能自动重连 | P1 |

### 1.2 详细用户故事

#### US-01: 建立 WebRTC 连接

```
作为用户，
当我打开语音助手网页时，
我希望系统自动建立 WebRTC 连接，
以便能够进行语音对话。
```

**验收标准**:
- [ ] 网页加载完成后自动尝试连接
- [ ] 连接成功时显示"已连接"状态
- [ ] 连接失败时显示错误信息并提供重试按钮

---

#### US-02: 语音对话

```
作为用户，
当我对着麦克风说话时，
我希望 AI Agent 能识别我的语音并回复我，
以便进行自然的语音对话。
```

**验收标准**:
- [ ] 麦克风权限请求后开始录音
- [ ] 说话停止后自动发送音频到服务器
- [ ] 收到 AI 回复后自动播放语音
- [ ] 对话延迟 < 5 秒

---

#### US-03: 状态显示

```
作为用户，
我希望能清楚看到当前系统的状态，
以便我知道是否可以说话。
```

**验收标准**:
- [ ] 显示"等待中"状态
- [ ] 显示"正在录音..."状态
- [ ] 显示"正在思考..."状态
- [ ] 显示"正在播放..."状态

---

#### US-04: 断线重连

```
作为用户，
当网络断开时，
我希望系统能自动重连，
以便对话不中断。
```

**检测条件**: WebRTC 连接断开
**重试策略**: 指数退避，3 次重试后提示用户

---
